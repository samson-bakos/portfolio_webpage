SnoopChat

Snoop Dogg AI Text-to-Speech Persona | April 2024

Key Skills: OpenAI, Google Cloud Platform, Audio Processing, Python, Streamlit

GitHub Link

App Link

SnoopChat is a text-to-speech application that allows users to interact with a virtual representation of Snoop Dogg. I undertook this project to develop my understanding of modern AI tools and technologies, including fine tuning of GPT models and AI Voice Synthesis. I selected Snoop Dogg as a subject due to the challenge of his unique voice and style of speaking.

Project Overview

Data Acquisition: Initial transcripts were sourced from SnoopDoggTV's YouTube segments, but it was challenging to get a clean transcript from YouTube’s automated captioning. To improve quality, I used a more detailed extended interview with Larry King, which provided clearer insights into Snoop's speech patterns.

AI Modeling: The data was processed and fine-tuned using GPT-3.5, focusing on accurately reflecting Snoop's unique vocabulary and style. While it is possible to achieve Snoop Dogg-like speech by prompting GPT-3.5 directly, I compared this output against actual transcripts of Snoop’s speech and found a non-fine-tuned model gave an overly embellished, stereotypical version of Snoop’s speech.

Voice Synthesis: I extracted audio clips from Snoop’s YouTube channel and used Google Cloud Platform’s voice transcription to extract timestamps for different speakers. I then extracted and concatenated the clips corresponding to Snoop’s speech to create a cohesive voice sample. I then trained a voice synthesis model with gemelo.ai, ensuring the virtual voice closely matched the real Snoop Dogg.

Interface Development: I developed a simple, user-friendly web interface with Streamlit, allowing easy interaction with the AI.

Challenges and Learning

The project highlighted the importance of quality data in AI training and the challenges of voice synthesis. The limited data used led to potential overfitting to the given prompts (~250 total), suggesting future improvements with access to more a diverse dataset of Snoop's speech.

SnoopChat is a work in progress. Try it live and share your feedback to help develop it further!

ChatPDF is a Streamlit-based web application that allows users to interactively query PDF documents. I undertook this project to develop my understanding of modern AI tools and technologies like the OpenAI API and LangChain.

Project Overview

Natural Language Querying: Integrates OpenAI's language models via LangChain to respond to user queries in natural language, allowing for efficient document exploration.

Predicting Floods in Canadian Rivers

BGC Engineering | MDS Capstone Project | April - July 2023

GitHub Link

This project focused on the prediction of flood magnitudes in Canadian rivers. While the flows of many rivers are actively monitored, many more are not. Estimates of maximum flow in these unmonitored rivers are necessary to safely design infrastructure in the surrounding areas.

My team focused on making improvements to the existing standard methodology, as outlined by the US Geological Survey. This methodology has multiple components, including grouping of similar rivers, fitting of distributions, and mean/quantile flow predictions. My team explored possible improvements to all aspects of the existing approach, using modernized machine learning and applied statistics to create a system that reduced error by 84% at the key design-flood threshold, as well as automating a time consuming manual step requiring expert domain knowledge.

Our system is currently in use by BGC's Engineers and Hydrologists to guide engineering decisions to keep Canadian communities safe. Our project was also selected as a 'Top 3 Best Project' by UBC MDS peers and faculty. You can read the full report at the link above.

Wordle Cheater

Statistical Solver for the Daily Wordle | July 2023

Key Skills: Python, Applied Statistics, Streamlit, NumPy, Git/Github, CSS

App Link

GitHub Link

My take on a solver for the NY Times Wordle Puzzle.

I created this project partially as an excuse to learn Streamlit as an intuitive platform for ML webapp development, but mostly because my fiance is much better at Wordle than I am.

It works by selecting the word with the highest joint probability of its letters being in the correct position from a list of remaining possible solutions, filtering the list and re-ranking solution probabilities based on user input at each step. Experimentation has found this implementation requires only 3.5 guesses per solve, compared to the global user average of 4.016.

I also spent time modifying the UI using custom CSS, and it is designed to mimic the layout of the traditional Wordle board, especially on a dark mode browser. Give it a try at the link above!

CompAssist

Video Game Probability Calculator | Feb 2023

Key Skills: Package/ Library Development, Python, R, SciPy, Applied Statistics, CI/CD

GitHub Link - Python Package

GitHub Link - R Library

I built this project for DSCI 524 - Collaborative Software Development. It was inspired by a real-life statistical problem I ran into during winter break in the UBC MDS. Many video games have multiple rewards from a single encounter, and many players set the goal to obtain all of them. It is trivial to calculate the number completions needed when all rewards have the same drop rate, or there are only a few, but when there are multiple rewards with different rates, the problem becomes much more complicated.

Specifically, I wanted to know the number of completions required to finish the Tombs of Amascut raid in Old School Runescape. There are seven rewards (with 5040 possible drop permutations), each having a different chance of being obtained. I developed some specific Python code, but realized that similar problems are present for other RuneScape bosses, as well as in many other videogames. DSCI 524 offered me a convenient opportunity to create a general solution as a Python Package/ R Library for public use.

Along with my teammates, we worked from this original idea to expand out the functionality of the package/library. In addition to the original calculation function, we added a modified binomial probability calculator to visualize how lucky/unlucky a user is, a method to select the optimal play method from multiple candidates, and a calculator specific for hunting 'Shiny' Pokemon. You can explore the apps in the links above, as well as using the publicly available versions on PyPi and CRAN.

Credit Default Predictor

Supervized Learning Classification System | Dec 2022

Key Skills: Python, SciKit-Learn, Machine Learning, Pandas, NumPy, Git/Github

GitHub Link

One of my earliest data science projects, I created this project during winter break after the first three blocks of the UBC MDS. After very quickly covering the fundamentals of supervized machine learning, I wanted to consolidate my understanding with an exploratory project.

I tried to cover as much ground as possible with this project, investigating multiple classification algorithms in parallel to create an optimized model to predict if a credit card user would default on next month's payment based on their past behavior, including both SciKit-Learn native algorithms, and other Boosted Tree based models (i.e. LightGBM). I also explored extra steps such as feature engineering/ feature selection/ feature importance interpretation, and performed fine tuning of hyperparameters on strong candidate algorithms. I assessed candidates using multiple classification criteria, including ROC/PR curves and confusion matrices, and thoroughly explained my thinking on the precision/recall tradeoff for this problem. Finally, I selected and evaluated a threshold modified random forest model on the test set, where it achieved 83% recall at a reasonable precision threshold

I now use this project as a self-constructed workflow documentation for classification problems, referring back it for inspiration and syntax when approaching similar problems in my data science practice. This workflow could also be easily adapted to a real life credit assessment context to improve risk management, increase assessment efficiency, and prevent loss of revenue. You can see the full document at the GitHub link above.

Data Scientist Intern

BGC Engineering | April - July 2023

Following the completion of my UBC MDS coursework, I worked with BGC Engineering to update their system for predicting floods in Canadian rivers. Our task was to predict the "One in Two Hundred Year Flood" in rivers where no annual flow data was available, but some characteristics of the watershed were known. This was a statistically challenging problem as it required not just a prediction of an expected value, as in a typical ML problem, but the prediction of the highest quantiles of an unknown distribution, using sometimes very limited data from 'similar' distributions.

Building upon existing approaches and under guidance from both MDS Mentors and BGC's experts, my team made improvements to both the machine learning and statistical components of the prediction process, ultimately creating a system that reduced prediction error by 84% at the key 0.995 quantile. Our pipeline script is currently in use by BGC Engineering in the backend of their proprietary software 'Cambio' to guide engineering decisions and keep Canadian communities safe.

Our project and final presentation was selected as a "Top 3 Best Project" by UBC MDS peers and staff, and was recieved extremely well by BGC's community of Engineers and Hydrologists following our company wide presentation.
